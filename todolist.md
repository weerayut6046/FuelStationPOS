## Acceptance Criteria ของระบบ

- [ ] **AC-01:** อินเทอร์เน็ตภายนอกขัดข้องแล้วยังขาย รับเงิน และพิมพ์เอกสารได้
- [ ] **AC-02:** เมื่ออินเทอร์เน็ตกลับมา ข้อมูลค้างซิงก์ครบโดยไม่สร้างยอดซ้ำ

- [ ] **AC-06:** ยอดปิดกะกระทบยอดกับการขาย การชำระเงิน และเงินสดได้
- [ ] **AC-07:** การยกเลิก คืนเงิน เปลี่ยนราคา และปรับสต็อกมีผู้อนุมัติและ Audit Log
- [ ] **AC-08:** การหลุดการเชื่อมต่อกับตู้จ่ายไม่ทำให้ธุรกรรมหายหรือซ้ำ
- [ ] **AC-09:** เว็บ 3D แสดงสถานะถูกต้อง และหาก 3D ใช้ไม่ได้ POS ยังทำงานตามปกติ
- [ ] **AC-10:** Backup ที่กำหนดสามารถ Restore ในเครื่องทดสอบได้จริง
- [ ] **AC-11:** ปิดวันธุรกิจแล้วแก้ย้อนหลังได้เฉพาะผู้มีสิทธิ์และมี Audit Log
- [ ] **AC-12:** ยอดรับชำระทุกช่องทางกระทบกับ Settlement/เงินสดและแสดงส่วนต่างได้
- [ ] **AC-13:** สต็อกน้ำมันตามบัญชีเทียบยอดวัดจริงได้และแจ้งเตือนส่วนต่างเกินเกณฑ์
- [ ] **AC-14:** รายการจัดซื้อย้อนกลับจาก PO ถึงรับเข้า Invoice และการคืนได้
- [ ] **AC-15:** ลูกค้าเครดิตถูกควบคุมด้วยวงเงิน เงื่อนไข และรายงาน Aging
- [ ] **AC-16:** ข้อมูลส่งออกบัญชีมี Mapping และยอดควบคุมตรงตามแบบอนุมัติ
- [ ] **AC-17:** Master Data และยอดยกมาผ่านการกระทบยอดและอนุมัติก่อน Cutover
- [ ] **AC-18:** เหตุผิดปกติสำคัญสร้าง Alert และมีผู้รับผิดชอบภายใน SLA
- [ ] **AC-19:** ข้อมูลแยกสิทธิ์ตามสาขาและรายงานส่วนกลางรวมยอดได้ถูกต้อง
- [ ] **AC-20:** ราคา/โปรโมชั่นที่กระจายมี Version เวลาเริ่มใช้ และสถานะตอบรับ
- [ ] **AC-21:** การสลับ 3D/2D คงสาขา ช่วงเวลา ตัวกรอง รายการที่เลือก และสิทธิ์ผู้ใช้
